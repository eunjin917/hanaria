### 커밋 규칙
- feat : 새로운 기능에 대한 커밋
- fix : 버그 수정에 대한 커밋
- refactor : 코드 리펙토링에 대한 커밋 + 파일 수정
- chore : 자잘한 수정에 대한 커밋(오타) + gitignore 관련 수정
- docs : 문서 수정에 대한 커밋
- test : 테스트 코드 수정에 대한 커밋
- config : 설정 파일에 대한 수정
- init: 초기 설정

### 브랜치 전략 : Git Flow
1. release(main)
2. develop
3. feature
